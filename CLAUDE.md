## Project Overview

The Gatekeeper Operator is a Kubernetes operator that installs and manages OPA Gatekeeper (Open Policy Agent) using the Operator Lifecycle Manager (OLM). It's part of the stolostron/Red Hat ACM ecosystem and manages the deployment of Gatekeeper for policy enforcement in Kubernetes clusters.

**Important:** The `main` branch contains only documentation. All operator code lives in release branches (e.g., `release-3.20`, `release-3.19`).

## Common Commands

### Build and Development

```bash
# Format and lint code
make fmt                    # Format Go code (gofmt, gofumpt, gci) and tidy go.mod
make lint                   # Run golangci-lint and YAML linting
make vet                    # Run go vet

# Build
make build                  # Build manager binary (runs generate, fmt, vet first)
make docker-build IMG=<image>  # Build container image (runs tests first)
make docker-push IMG=<image>   # Push container image

# Use podman instead of docker
make build DOCKER=podman
```

### Testing

```bash
# Unit tests
make test                   # Run all unit tests with envtest
make test-unit              # Run only unit tests
make test-coverage          # Generate coverage report (coverage_unit.out)

# E2E tests (requires cluster)
make test-cluster           # Create local kind cluster with registry
make kind-bootstrap-cluster IMG=localhost:5000/gatekeeper-operator:test  # Build, load, and deploy to kind
make test-e2e NAMESPACE=gatekeeper-system  # Run e2e tests
make test-gatekeeper-e2e    # Run Gatekeeper's own e2e test suite
make delete-test-cluster    # Clean up kind cluster

# OpenShift-specific tests
make test-openshift-setup
make test-e2e-openshift

# Run a single test
go test ./controllers -run TestGatekeeperController
```

### Running the Operator

```bash
# Install CRDs
make install

# Run locally (outside cluster)
make run NAMESPACE=gatekeeper-system

# Deploy to cluster
make deploy IMG=<image> NAMESPACE=gatekeeper-system
make undeploy

# Deploy using OLM
make deploy-olm             # Install OLM into cluster
make deploy-using-olm NAMESPACE=gatekeeper-system
```

### Code Generation

```bash
make manifests              # Generate WebhookConfiguration, ClusterRole, CRDs
make generate               # Generate DeepCopy methods for types
make import-manifests       # Import Gatekeeper manifests from stolostron/gatekeeper
make update-bindata         # Update bindata.go with embedded manifests
make update-gatekeeper-image  # Update Gatekeeper image reference
make release                # Generate release manifest (deploy/gatekeeper-operator.yaml)
```

## Architecture

### High-Level Structure

The operator follows the Kubernetes operator pattern with three main reconciliation loops:

1. **Gatekeeper Controller** (`controllers/gatekeeper_controller.go`)
   - Main reconciliation loop for the `Gatekeeper` CR
   - Deploys/manages three core Gatekeeper components:
     - Audit deployment (scans existing resources for violations)
     - Webhook deployment (validates admission requests)
     - Controller manager deployment (manages constraints and templates)
   - Creates CRDs, RBAC, webhook configurations, and services
   - Uses embedded manifests from `pkg/bindata/` (sourced from stolostron/gatekeeper)
   - Applies strategic merge patches to allow user customization

2. **Config Controller** (`controllers/config_controller.go`)
   - Manages the default Gatekeeper `Config` resource
   - Sets default exempt namespaces (kube-*, openshift-*, etc.)
   - To modify default exempt namespaces, edit `controllers/config_helper.go:getDefaultConfig()`

3. **ConstraintStatus Controller** (`controllers/constraintstatus_controller.go`)
   - Aggregates constraint violation status across the cluster
   - Collects data from individual pod status objects

### Key Directories

- **api/v1alpha1/** - `Gatekeeper` CRD definition and types
- **controllers/** - Reconciliation logic for all three controllers
- **pkg/bindata/** - Embedded Gatekeeper manifests (generated by go-bindata)
- **pkg/platform/** - Platform detection (OpenShift vs vanilla Kubernetes)
- **config/gatekeeper/** - Imported raw Gatekeeper manifests (source of truth)
- **config/samples/** - Example Gatekeeper CR configurations
- **test/e2e/** - End-to-end test suites using Ginkgo

### Resource Management

The operator uses a strategic merge pattern:

1. Starts with base Gatekeeper manifests from `pkg/bindata/`
2. Applies user-specified overrides from the `Gatekeeper` CR spec
3. Uses `controllers/merge/` package for merging logic
4. Supports customization of: replicas, resources, affinity, tolerations, nodeSelector, image, logLevel, etc.

## Version Files

- **VERSION** - Operator version (e.g., 3.19.1)
- **GATEKEEPER_VERSION** - Gatekeeper version to deploy (e.g., 3.19.3) - may differ from operator version
- **REPLACES_VERSION** - Previous operator version for OLM upgrade path
- **go.mod** - Go 1.23+ required

When updating Gatekeeper version:

```bash
echo "3.20.0" > GATEKEEPER_VERSION
go get github.com/open-policy-agent/gatekeeper/v3@v3.20.0
go mod tidy
make import-manifests update-bindata update-gatekeeper-image release bundle
```

## Testing Strategy

1. **Unit Tests** - Test individual controller logic with envtest (fake Kubernetes API server)
2. **E2E Tests** - Deploy full operator to kind cluster, create CRs, verify deployments
3. **Gatekeeper E2E** - Run upstream Gatekeeper's BATS test suite to verify manifests
4. **OLM Tests** - Verify bundle installation via OLM

All PRs run: fmt/lint checks, unit tests, E2E tests, OpenShift E2E tests, Gatekeeper E2E tests, and gosec security scanning.

## Important Notes

- The operator uses `go-bindata` to embed Gatekeeper manifests at build time
- After modifying `config/gatekeeper/` manifests, always run `make update-bindata`
- The operator detects OpenShift vs vanilla Kubernetes and adjusts SecurityContext accordingly
- Default exempt namespaces prevent Gatekeeper from validating system namespaces
- ValidatingWebhookConfiguration is enabled by default; set `spec.validatingWebhook: DISABLED` to disable
- Use `config/samples/` as starting points for Gatekeeper CR configurations
